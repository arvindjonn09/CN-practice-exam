Security Principles: CIA, Threat, Vulnerability, Exploit and Risk

This is the first real cybersecurity language lesson. Before tools, firewalls and logs, you must be able to separate what is valuable, what can go wrong, how it can be attacked, and what business impact it creates.

# Simple Explanation

CIA Triad

  • Confidentiality: only the right people can see the data.
  • Integrity: data stays correct and is not secretly changed.
  • Availability: systems and data are usable when needed.

Threat

  • A possible danger that can cause harm.
  • Example: attacker, malware, insider, power failure, phishing email.

Vulnerability

  • A weakness that can be used by a threat.
  • Example: weak password, unpatched software, open port, bad permission.

Exploit

  • The method or action used to take advantage of a vulnerability.
  • Example: using a known bug to run code or stealing credentials through phishing.

Risk

  • The chance and impact of something bad happening.
  • Risk = valuable asset + threat + vulnerability + possible impact.

Hardening

  • Reducing weak points before an attack.
  • Example: patching, disabling unused services, strong passwords, MFA and least privilege.

# Real-World Example

Scenario: a small office stores customer invoices on a shared Windows folder. One user has a weak password and the folder allows too many people to edit files.
TermIn this scenario
AssetCustomer invoices and business records.
ThreatPhishing attacker or insider misuse.
VulnerabilityWeak password and excessive folder permissions.
ExploitAttacker steals the password and modifies or deletes invoices.
RiskData loss, wrong billing, privacy breach, business downtime.
HardeningMFA, strong password policy, least privilege, backups and audit logs.

# Interview Answer Builder

Answer: The CIA triad is the basic model for protecting information. Confidentiality means only authorized people can access data, integrity means data remains accurate, and availability means systems are usable when needed. A threat is something that can cause harm, a vulnerability is a weakness, an exploit is the method used to abuse that weakness, and risk is the possible business impact. In a support role, I would identify the asset, check the weakness, apply controls like MFA, patching and least privilege, then document the evidence.
