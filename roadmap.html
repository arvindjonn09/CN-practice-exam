# How Every Topic Must Work

1. Learn

  • Where to study from.
  • Simple explanation and real-world example.
  • Diagram or visual model.

2. Lab

  • Build the topology.
  • Configure the feature.
  • Verify with commands.

3. Break / Fix

  • Create one realistic mistake.
  • Record symptom, root cause and fix.
  • Save comments and confusion.

4. Prove

  • Upload screenshot or video.
  • Export note.
  • Save GitHub evidence path.

# Proof Standard

A topic is not complete until someone else can understand what was studied and what was proven.

Proof TypeWhat it shows
NotesWhat I understood in my own words.
ScreenshotVisible result: topology, command output, log, configuration or setting.
VideoProcess proof: how I checked, tested, broke and fixed something.
GitHub pathPermanent proof location for labs, configs, notes, screenshots and mistakes.
CommentsConfusion, mistake, review reminder and next improvement.
